Command-line Options Supported in Single-Run and Multi-Run MSIE Flows

The following options are supported both in Single-step and Multi-step MSIE flows.

OptionSingle-xrun MSIE Usage DescriptionMulti-xrun MSIE Usage Description

-incrtop module_name

For primary instantiating incremental flow, -incrtop <module_name> is needed.

Identifies the top level of the incremental partition when using the -genhref flow to generate a hierarchical permission file.

The -genhref option generates a hierarchical permission file before creating a primary snapshot. See Generating an href File. To identify unbound modules in the incremental partition, add the -incrtop option to the command line.
-msie_verbose

Includes information in the elaboration output and log file about what primary snapshots were used and to what instances they were bound.

 Includes information in the elaboration output and log file about what primary snapshots were used and to what instances they were bound.

-msietopsok

Allows identically named top-level primaries when building the simulation snapshot.

When a primary snapshot is included in a simulation snapshot, and that primary snapshot has some top-level design units that are not instantiated within the hierarchy, these uninstantiated top-level units become top levels in the simulation snapshot. If a simulation snapshot has more than one top-level design unit with the same name, then the simulator attempts to merge the duplicate top-level modules automatically.

Only those duplicate top modules that are not instantiated within the hierarchy are merged. Otherwise, the simulator generates an error when attempting to merge multiple copies. Use this option to change the error to a warning and allow multiple instantiated top-level design units with the same name to co-exist.

 Same behavior as in single xrun MSIE flow.

-primtop module_name

Single xrun MSIE flow is enabled with -primtop and -incrtop options.

For example:

% xrun -f dutsrc.f tb_top.v test1.v -primtop dut -primtop dut_sub \
 -primparallelelab 2 -top tb_top

Identifies the top level of the primary partition when using the -genhref flow to generate a hierarchical permission file.

The -genhref option generates a hierarchical permission file before creating a primary snapshot. See Generating an href File. To identify unbound modules in the primary partition, add the -primtop option.

For example:

% xrun -f dutsrc.f tb_top.v test1.v -genhref href.txt -primtop dut
-primbind Used for applying verilog configuration rules in single xrun MSIE flow.

Used for loading all the primaries lying in same xmlibdirname.

-primbind will mean all -primname <primary_snapshot_name>
-msie_href_verbose Provides verbosity related href and extern file. Same behavior as in single-xrun MSIE flow.
