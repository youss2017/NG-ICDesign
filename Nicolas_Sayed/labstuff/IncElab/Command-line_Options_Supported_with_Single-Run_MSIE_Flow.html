Command-line Options Supported with Single-Run MSIE Flow

The following commands are supported with the Single-run MSIE flow.

OptionDescription
-makeprim <OPTIONS>-endprim Specifies the end of the list of top-level design units and options for a primary partition.
-libverbose

This option prints verbose messages about instance binding.

By default -libverbose option is not enabled on MSIE genhref stage in single xrun MSIE flow. You can enable this option by specifying -enable_libverbose_genhref on command-line.

-nopreelab

Skips the initial step of analyzing the design for hierarchical references and primary parameter values. This saves some time, but it then becomes your responsibility to manage hrefs and parameterization of primaries.

When specifying this option, take care when naming and using primary partitions.
-prefix_primelab <User_DRM_Script>

When you run xrun in the single-step mode, you can run MSIE with any DRM (distributed resource management) system to distribute the primary builds when you have multiple primary partitions.

-prefix_primelab specifies a DRM command or script that distributes the builds on the DRM system. You must also specify -primparallelelab, to run in single-xrun parallel mode. See Running xrun in Single-Step Mode.

-primparallelelab number

Elaborates primary partitions in parallel. Use this option to specify the number of parallel primary elaborations to run.

You can also specify the number of primary parallel elaborations to run using the PRIMMAXPARALLEL environment variable. When specifying both the environment variable and the command-line option, the environment variable takes precedence.
-msie_verbose Prints information about MSIE partition boundaries. The -msie_genhref_verbose option provides href and OOPR verbosity.
-xmsie_verbose

Prints messages about verbosity automatic partitioner.

The -xmsie_verbose option is enabled by default under -AUTOMSIE option.

By default, the -xmsie_verbose option is not enabled with -GENPARTITION option. To have partitioner verbosity with -GENPARITION option, -xmsie_verbose option is required.
-xmsiepartitiontop <BLOCK_NAME> Specify top level of the hierarchy to be partitioned. Generally, this option is used for re-running the partitioner for breaking the long pole primary.
-xmsienumpartitions Specifies number of partitions for MSIE partitioner.
-xmsienumclonepartitions Specifies the cloning limit for MSIE partitioner.
-xmsieclonedonly Used with -automsie to generate only replicated partitions.
-xmsiepkgsweight Includes SystemVerilog package weight in AutoMSIE partitioning
-xmsieperfstat Prints performance status of the AutoMSIE.
-gdbelab Runs xmelab under gdb.
-gdbhref Runs -genhref under gdb.
-gdbprim Runs primary builds under gdb.
-msieunlock all

Enables new MSIE features like complex OOPR cases, Verilog configurations, and so on.

With -AUTOMSIE option the -msieunlock_all option is enabled by default.
-msieunlock simperf

Enables simulation performance optimizations in MSIE flow. Enabling this option may elongate the build time by increasing the genhref (Analyzing MSIE Partition) phase time. This is due to the reason that the tool has to do more deeper simulation performance analysis and dump data and then apply to respective primaries.

With -AUTOMSIE option, the -msieunlock simperf option does not get enabled by default.
-href <href_file> Uses the given href file for the primary snapshot. Applicable in only -nopreelab.
-hrefforce Force href file in single xrun MSIE flow, especially in nopreelab flow where we want to avoid genhref elbaoration phase.
-primbind To apply verilog configuration in single xrun MSIE flow.
-enable_forward_class Enables SystemVerilog forward class  declaration (typedef). 
-enable_smart_cuscope Enabled new smart global compilation unit scope features. This breaks the cuscope into two parts: one for cuscope binds and another for others cuscope declarations. Also, with this feature, primaries which are not dependent upon any cuscope,  dot not get $unit content and thus scratch and incremental build time gets reduced.  W/o this feature, primary (irrespective of cuscope dependency), gets entire $unit content.
-showcudep/-cudep_file <arg_log file>

This option when used with -enable_smart_cuscope option helps to find out the different primaries/incremental dependencies to single or multiple global compilation units ($unit). It explains why a certain cuscope was loaded on a specific primary with the corresponding module (belonging to that particular primary) using cuscope target/source information.  

This option can also be run along with parser option -parseinfo cuscope_verbose.
