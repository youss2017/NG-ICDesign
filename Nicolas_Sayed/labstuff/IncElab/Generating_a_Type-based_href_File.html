Generating a Type-based href File

Type-based href file is a superset of href file. Each line in type-based file is unique. You can generate a type-based href file using the -gentypehref option.

The use model remains the same as for any other href file creation. The only enhancement is to allow:

  • Dump type-based information in -autohref flow
  • Dump type-based information in -genhref flow
  • Dump type-based information while using RENAME_AUTOHREF option

In certain scenarios, it is possible for multiple instances of a module to have identical permission markings for their signals. This can result in an instance-based href file with a large number of lines. However, utilizing a type-based href can reduce the number of lines by condensing duplicates. This can simplify the process of comprehending and interpreting the href files.

In some cases, it may happen that a particular signal of a module has forced only at one instance and is not read at all other places. In that case, the signal of all those module instances will be forced, causing huge simulation performance impact. Hence, in case of simulation performance benchmarking, the tool-generated instance-based href file is recommended for the particular incremental test.

Example 1:

//dut.sv 
module dut();
sub sub();
sub sub1();
sub sub2();
sub sub3();
endmodule
module sub();
reg a1,a2,a3 ;
endmodule

//t1.sv 
module test();
dut dut();
initial force test.dut.sub1.a1 =1'b0;
endmodule

//t2.sv
module test();
dut dut();
initial force test.dut.sub2.a1 =1'b0;
endmodule

-Autohref

Use the following command to run the primary build: 

xrun dut.sv -top dut -mkprimsnap -name dut -clean

Use the following command to run the incremental build:

xrun -primname dut t1.sv -top test -gentypehref

The tool reports a fatal error HREFIF and generates autohref.txt and autohref_type.txt

$ cat autohref.txt

#VERSION : TOOL: xmelab 23.02-a071-20230215

#TIME : 2023-2-15 22:43:6

dut.sub1.a1 F


$ cat autohref_type.txt

#VERSION : TOOL: xmelab 23.02-a071-20230215

#TIME : 2023-2-15 22:43:6

sub a1 F


The testcase will successfully run using the below commands:

xrun dut.sv -top dut -mkprimsnap -name dut -clean -href autohref_type.txt
xrun -primname dut t1.sv -top test
xrun -primname dut t2.sv -top test

-Genhref

Use the following command to run the primary build: 

xrun dut.sv -top dut -mkprimsnap -name dut -clean -gentypehref

Use the following command to run the incremental build:

xrun -primname dut t1.sv -top test -genhref href.txt -gentypehref

The tool reports a fatal error HREFIF and generates href.txt and href_type.txt

$ cat href_type.txt

#VERSION : TOOL: xmelab 23.02-a071-20230215

#TIME : 2023-2-15 22:47:12

sub a1 F

The testcase will successfully run using the below commands:

xrun dut.sv -top dut -mkprimsnap -name dut -clean -href href_type.txt
xrun -primname dut t1.sv -top test
xrun -primname dut t2.sv -top test

-RENAME_AUTOHREF

Use the following command to run the primary build: 

xrun dut.sv -top dut -mkprimsnap -name dut -clean

Use the following command to run the incremental build:

xrun -primname dut t1.sv -top test -RENAME_AUTOHREF abchref.log -gentypehref


The tool reports a fatal error HREFIF and generates abchref.log and abchref_type.log

$ cat abchref_type.log

#VERSION : TOOL: xmelab 23.02-a071-20230215

#TIME : 2023-2-15 22:55:12

sub a1 F

The testcase will successfully run using the below commands:

xrun dut.sv -top dut -mkprimsnap -name dut -clean -href abchref_type.log
xrun -primname dut t1.sv -top test
xrun -primname dut t2.sv -top test

Example 2:

//dut.sv
module dut();
sub sub();
sub sub1();
sub sub2();
sub sub3();
endmodule
module sub();
reg a1,a2,a3 ;
endmodule

//t1.sv
module test();
dut dut();
initial force test.dut.sub1.a1 =1'b0;
initial force test.dut.sub1.a2 =1'b0;
initial force test.dut.sub2.a2 =1'b0;
initial $display ("%b",test.dut.sub1.a2);
endmodule

Use the following command to run the primary build:

xrun dut.sv -top dut -mkprimsnap -name dut -clean

Use the following command to run the incremental build:

xrun -primname dut t1.sv -top test -GENTYPEHREF

The tool reports a fatal error HREFIF and generates autohref_type.txt and autohref.txt

Below is a log of output generated:

$ cat autohref.txt

#VERSION : TOOL: xmelab 23.02-a071-20230216

#TIME : 2023-2-16 23:45:24

dut.sub2.a2 F

dut.sub1.a1 F

dut.sub1.a2 RF


$ cat autohref_type.txt

#VERSION : TOOL: xmelab 23.02-a071-20230215

#TIME : 2023-2-15 23:9:47

sub a2 RF

sub a2 F

sub a1 F

The testcase will successfully run using the below commands:

xrun dut.sv -top dut -mkprimsnap -name dut -clean -href autohref_type.txt

xrun -primname dut t1.sv -top test

Limitations

  • This feature is not supported in Single-run MSIE flow.
  • Currently, it does not support -genmsiedata flow.
