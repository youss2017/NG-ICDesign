Product Version 23.09, September 2023

Generating an href File When Elaborating the Simulation Snapshot

When the simulation snapshot is elaborated, and there are missing permissions in a primary, the elaborator generates HREFAC messages indicating what permissions are missing. The elaborator then automatically generates an href file.

The generated href file is called autohref.txt. If this file exists, it generates autohref1.txt, then autohref2.txt, and so on.

If the elaborator cannot open the file (for example, because of permission problems in the working directory), the elaborator outputs the required permission text to stdout and to the log file.

The autohref.txt file can then be included with the -href option when you create the primary snapshot. For example:

% xrun primary.v -mkprimsnap -snapshot dut
...
module worklib.dut:v
...
Writing primary snapshot: worklib.dut:v
% xrun incremental.v -primname dut
...
module worklib.tb:v
...
Elaborating the design hierarchy (incremental partition):
...
[MSIE] Instance tb.dut_inst is bound to module worklib.dut:v in primary snapshot worklib.dut:v
...
initial $display(dut_inst.w);
|
xmelab: *N,HREFAC (./incremental.v,5|28): Hierarchical reference to primary partition requires permissions.
The referenced object needs permission to be read.
Wrote HREF file: autohref.txt
Primary models that need to be re-elaborated:
worklib.dut:v
# Elaborate primary snapshot with the href file.
% xrun primary.v -mkprimsnap -href ./autohref.txt -snapshot dut
# Elaborate incremental snapshot and simulate.
% xrun incremental.v -primname dut
Or you can use the -primhrefupdate option to automatically update missing permissions in a primary using the autohref.txt file during incremental elaboration. This option is available for direct invocation and multi-step xrun mode. For example:

Direct Invocation Mode

# Compile the source files for the design.
% xmvlog primary.v incremental.v
# Create the primary snapshot.
% xmelab -mkprimsnap dut
# Update any missing permissions and create the simulation snapshot.
% xmelab tb -primbind -primhrefupdate -messages
Loading primary snapshot worklib.dut:module .................... Done
Elaborating the design hierarchy (incremental partition):
...
Loading primary snapshot worklib.dut:module .................... Done
[MSIE] Instance tb.dut_inst is bound to module worklib.dut:module in primary snapshot worklib.dut:module
initial $display(dut_inst.w);
|
xmelab: *N,HREFAC (./incremental.v,5|28): Hierarchical reference to primary partition requires permissions.
The referenced object needs permission to be read.
Wrote HREF file: autohref.txt
Primary models that need to be re-elaborated:
worklib.dut:module
Updating primary snapshot worklib.dut:module (PRM), reason: New MSIE href permissions are needed.
...
Update of primary snapshot worklib.dut:module (PRM) successful.
...

Multi-Step xrun Mode

# Create the primary snapshot.
% xrun primary.v -mkprimsnap -snapshot dut
# Build the incremental, update any missing permissions, and run the simulation.
% xrun incremental.v -primname dut -primhrefupdate
