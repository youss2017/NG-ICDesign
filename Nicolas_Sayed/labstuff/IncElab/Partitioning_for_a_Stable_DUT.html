Partitioning for a Stable DUT

In this partitioning style, the stable DUT is elaborated into the primary snapshot. The entire testbench, which instantiates the DUT, and which is being developed and debugged, is elaborated into the incremental snapshot.

This type of partitioning can reduce the re-elaboration time spent whenever the testbench is modified. The re-elaboration time gain is the time it takes to elaborate the DUT.

A common application of this partitioning style is when a large gate-level netlist is simulated with several ATPG generated scan tests.

Advantages of Partitioning a Stable DUT

Partitioning a stable DUT has several advantages:

  • It can be applied in almost every environment, as there is always an instantiation of DUT that can be easily identified and elaborated independently into the primary snapshot.
  • The use model is intuitive, and the partitioning requires minimal effort.
  • It can be applied to complex testbenches.
  • It offers maximum flexibility in modifying the testbench.
  • It works with UVCs.

The testbench cannot override DUT parameters. The DUT must be compiled with final parameter values.
