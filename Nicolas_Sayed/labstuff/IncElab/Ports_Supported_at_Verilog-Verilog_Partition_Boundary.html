Ports Supported at Verilog-Verilog Partition Boundary

If the partition boundary is Verilog-Verilog, the following apply:

  • Port Type:

    Supported ScenariosUnsupported Scenarios
    Nets and variables of type wire/reg real
    Interface Ports Multi-dimensional unpacked array port expressions
    PART_SELECT and BIT_SELECT expressions on formal ports for all existing supported datatypes in Incremental instantiates primary flow Complex expression on actual ports
    Concatenation expression on actual ports The elaborator generates a DSSUFL error if unsupported port connection types are used.
    Concatenation expression on formal ports when net/wire/tri types are used for port connection in IIP (incremental-instantiates-primary)/PIP (primary-instantiates-primary) and PII (primary-instantiates-incremental) scenarios. The support for reg/var/logic type in IIP mode is available in both single-step and multi-step runs.

    The support for reg/var/logic type in PIP/PII mode is limited to single-step xrun only.

  • Port Map:

    Supported ScenariosUnsupported Scenarios
    Simple names, part-selects, bit-selects, constants, open ports, and unconnected ports A one-dimensional array (memory)
    Named mapping and positional mapping Nets with wire delays cannot be used in a port map at the partition boundary.
    The elaborator generates an INCDNS error if delays are specified.

    Dot-star (".*) wildcard expression in PIP (primary-instantiates-primary) and PII (primary-instantiates-incremental) scenarios

    The support of .* is not available in multi-step flow.


Tran Network Restrictions
If a boundary port is part of a tran network in more than one partition, then that boundary port must be declared with the same wire type in all partitions.
