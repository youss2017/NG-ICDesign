Ports Supported for VHDL-VHDL Partition

If the partition boundary is VHDL-VHDL, the following apply:

  • Port type

    Supported Scenarios at Partition boundaryUnsupported Scenarios

    std_logic

    		 integer, bit, bit_vector

    std_logic_vector

    		 boolean, time, string, character

    std_ulogic

    Floating point

    std_ulogic_vector

    		 Enumerated type

    real

    		 Physical type

    records (sub-element as std_logic, std_ulogic, std_logic_vector, std_ulogic_vector)

    Array of records

    		Access and file type

    User-defined types and sub-types


    The elaborator generates an INCUNS error if unsupported port connection types are used at the partition boundary.
  • Port map
    • Named mapping and positional mapping are supported.
    • Simple names, array elements, array slices, constants, literals (string literals, enumeration literals, aggregate literals), open ports, and unconnected ports are supported.
    • Buffer ports at the partition boundary are not supported. A DSSBFF error is generated if buffer ports are used at the boundary.
    • A signal declared in a package cannot be used in a port map expression. If a signal declared in a package is used for a port map between primary and incremental, the elaborator generates an INCPKG error.
    • A component port must be associated as an actual with at least one formal.
    • The size of a port expression must match the size of the port declaration at the snapshot boundary. A DSSMSM error is generated if the sizes do not match.
    • Guarded signals and ports are not supported at the partition boundary.
  • Type conversion functions are supported in a port map with the following restrictions:
    If a type conversion involves a VHDL port of type INOUT, then:
    • The argument of the type conversion and the return type of the type conversion must be one of the supported port types listed above, or the elaborator generates an INCUNS error.

         xmelab: *F,INCUNS: [inc-elab] Expression not supported in formal or actual of port connection.

    • Connections between unresolved and resolved types are not allowed. The elaborator generates an INCRST error if an unresolved type is connected to a resolved type.

         xmelab: *F,INCRST: [inc-elab] Resolved type object in incremental is connected to unresolved type object in primary

    • Boundary ports cannot be mapped to an alias. The simulator generates a DSSUNS error.

For VHDL, if the primary instance is bound with the incremental using direct Instantiation, the elaborator generates a DYNVHD2 error.

xmelab: *E,DYNVHD2: Overall design has multiple VHDL tops.
