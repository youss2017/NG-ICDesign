Running MSIE with xrun

Generally, xrun simplifies the invocation process by using one command line for compiling, building, and simulating a design. However, using xrun for MSIE can be more involved.

The following two use models are supported:

Mode of runningDescription
Single-step mode

With single-step invocation mode, you use one xrun command to build and simulate a complete MSIE model, and you are no longer required to specify multiple xrun commands when building each partition. See Running xrun in Single-Step Mode.
Multi-step mode

With multi-step invocation mode, you use separate xrun commands to build each partition, and the right set of options must be specified for each build command. The final xrun command simulates the complete MSIE model. See Running xrun in Multi-Step Mode.

Besides describing these two modes, this topic describes:

  • "Building Primaries in Different Directories"
  • "Running in Separate Directories"
  • "Source File Inter-dependency"

Building Primaries in Different Directories

You can use the -xmlibdirname option to build a primary snapshot in a different directory than the directory you use to build the final snapshot. When you do so, the xrun command used to elaborate the simulation snapshot must do one of the following:

  • Use the same -xmlibdirname option
  • Specify the same directory using the -primlibdir option
  • Specify the same directory using the -primname option

For example:

% xrun tb_top.v -incrbind test -name tb_top -xmlibdirname dir1
% xrun test.v -primname tb_top -xmlibdirname dir1

or:

% xrun test.v -primname tb_top -primlibdir dir1

or:

% xrun test.v -primname tb_top@dir1

Running in Separate Directories

The xrun command can also be run in separate directories. Use either the -primname or -primlibdir option on the command line to specify the location of the primary snapshot.

For example:

% cd ./primdir
% xrun tb_top.v -incrbind test -name tb_top
% cd ..
% xrun -primname tb_top -primlibdir ./primdir test.v

or:

% xrun -primname tb_top@./primdir test.v

Source File Inter-Dependency

Verilog source files can have an ordering dependency because of macros and directives (such as `timescale) that carry over from one file to the next when all files are parsed with the same command invocation. When using xrun with MSIE, however, source files are compiled in steps: the source files for each primary partition are passed to one xrun command and the source files for the incremental partition are passed to another xrun command. Macro definitions and directives may not carry over from one file to the other as if they were all parsed with the same command.

When converting an existing single xrun invocation build script to use MSIE, take care not to alter the model because of changes to the order in which the Verilog source is parsed. In general, construct the source files so that there are no source file inter-dependencies due to macros or directives between partitions.
