Running xrun in Single-Step Mode

For single-step xrun invocation mode, the software takes care of all parameter overrides internally. For instance, if the same primary snapshot is instantiated multiple times with different parameter values, xrun automatically creates different versions of that primary snapshot by identifying the combination of parameter values specified and passing these values on to separate configurations.

See Customizing Primary Snapshots.

MSIE allows parameters of string datatype at partition boundary.

Run xrun to compile, elaborate, and simulate a complete MSIE model.

The following options provide partition boundary information:

-primtop Specifies a top level of a primary partition.
-incrtop Specifies a top level of the incremental partition.
-makeprim Specifies the beginning of a list of top-level design units and options for a primary partition.
-endprim Specifies the end of the list of top-level design units and options for a primary partition.
-nopreelab

Skips the initial step of analyzing the design for hierarchical references and primary parameter values. This saves some time, but it then becomes your responsibility to manage hrefs and parameterization of primaries.

When specifying this option, take care when naming and using primary partitions.
-primparallelelab

Elaborates primary partitions in parallel. Use this option to specify a number of parallel primary elaborations to run.

You can also specify the amount of primary parallel elaborations to run using the PRIMMAXPARALLEL environment variable. When specifying both the environment variable and the command-line option, the environment variable takes precedence.

Syntax:

% xrun files [-primtop primary][-primtop ...]|
  [{-makeprim primary <other_options> -endprim}][{-makeprim ...}]\
  [-incrtop incremental] \
  [-nopreelab] \ [-primparallelelab number]

For single-step invocation mode, the -primtop or -makeprim command-line options are required when specifying the top-level module or entity of a primary partition. The following example specifies the dut as the top of the primary partition:

% xrun dut.v tb_top.v -primtop dut

or

% xun dut.v tb_top.v -makeprim dut <other_options> -endprim

To create a model where the primary partition instantiates the incremental partition, use the -incrtop option. For instance, when simulating an MSIE model, a test program can be included that is instantiated in tb_top along with a dut module. With the following command, the -incrtop option explicitly specifies the test program as the top of the incremental:

% xrun dut.v tb_top.v testprog.v -primtop tb_top -incrtop testprog

Top-level modules that are not explicitly put in the primary with a -primtop option, as tb_top is above, are in the incremental partition.

Multiple primary partitions are allowed. To specify multiple primaries, use multiple -primtop options or specify multiple -makeprim collections as shown:

% xrun ip1.v ip2.v soc.v tb.v \
 -primtop soc -primtop ip1 -primtop ip2

or:

% xrun ip1.v ip2.v soc.v tb.v \

  -makeprim soc -endprim \
  -makeprim ip1 -endprim \
  -makeprim ip2 -endprim

Only one main top-level module is allowed within a -makeprim collection. If other top-level modules are required to be in the same primary snapshot, use the -top option within the same collections to add them.

Typically, there is little need to put multiple tops into primaries when using the single-step xrun flow. However, there are some cases where an additional top module may need to be forced into a given primary. For instance, there may be modules that have additional objects needed by that primary. These top modules are ancillary to the primary, which means that they cannot fulfill an instance in another partition and therefore become top-level modules in the overall design.

For example, the xrun command below adds the top-level module myglobals to the top-level primary soc:

% xrun ip1.v ip2.v soc.v tb.v \

  -makeprim soc -top myglobals <other_options> -endprim \
  -makeprim ip1 <other_options> -endprim \
  -makeprim ip2 <other_options> -endprim

If the -incrtop option is used within a -makeprim collection, then it applies only to instances of the given module within that primary.
