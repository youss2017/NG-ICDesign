Specifying External Access Permissions

In addition to the hierarchical reference permissions, a hierarchical reference from the incremental partition to an object in a primary partition may also require setting external access permissions (read, write, or connectivity access) on its target object. These external access permissions are the same as the permissions granted with the elaborator -access option. For example, the xm_mirror function requires write access on its target.

Because the characters R, E, W, and I are used for specifying hierarchical reference permissions, the first available letter from the words read, write, and connectivity are used as the character for specifying external access permissions (reAd, wriTe, Connectivity).

A hierarchical reference may also require a vector net to be expanded (to be represented as individual bit nets rather than as a single vector net).

To specify external access permissions in an href file use the characters shown in the following table.

PermissionCharacter
Read A
Write T
Connectivity C
Expand a vector X

Example (href File):

top.a.b.my_net RDX      # Read and Drive permission to top.a.b.my_net.
                        # Expand this vector net.
top.a.b.my_var DFE      # Drive, Force, and Event Control permission to
                        # top.a.b.my_var
top.a.b.my_task IB      # Invoke and Disable permission to top.a.b.my_task
top.a.d ... X           # Expand all vector nets in and below top.a.d
top.a.c *               # Full permission to all objects in scope top.a.c.
                        # Expand all vector nets in this scope.
top.a.d<WIRE> R         # Read permission to all wire objects in scope top.a.d
top.a.d ...<REG> RW     # Read and Write permission to all reg objects in
                        # scope top.a.d and below
top.a ...2 *            # Full permission to top.a till depth 2

Using msie_href_verbose

The msie_href_verbose option provides all hierarchical references present in href_internal.txt file and OOPRs in extern modules with additional information like file name and line number for debugging. 

Example (href_internal.txt file):

#VERSION : TOOL:        xmelab  20.05-a001-20200513
#TIME : 2020-5-14 0:40:11
# /extern_dump/test2_port.sv : 12
# /extern_dump/test2_port.sv : 13
top.a EDC
# /extern_dump/test2_port.sv : 14
dut.b_dut EDC

Here “top.a” contains debug information of all of its occurrences in design. This debug information is printed under the msie_href_verbose option.
