Multi-Snapshot Incremental Elaboration
Product Version 23.09, September 2023

Specifying Hierarchical Reference Permissions

Each object in a primary partition that is the target of a hierarchical reference from the incremental partition must be given special permission when the primary snapshot is elaborated. If the necessary permissions have not been granted, the elaborator generates an error. 

These hierarchical reference permissions are provided through an href file or through command line. An href file is a text file that specifies reference permissions of objects in a primary snapshot. The href file is included with -href option. It consists of lines that start with a hierarchical name followed by one or more characters that represent the type of permission being given.

In the following code, the incremental refers to a variable in the primary (primary.status). The elaborator generates an HREFAC message during elaboration of the incremental snapshot if the variable has not been given read permission.

primary.v
// Primary top module
module primary();
reg [3:0] status;
  incremental incr();
endmodule
incremental.v
// Incremental top module
module incremental;
  wire [3:0] status;
  assign status=primary.status;
endmodule

# Compile primary.v and create primary snapshot
% xrun primary.v -mkprimsnap -incrbind incremental -snapshot top
file: primary.v
        module worklib.primary:v
                errors: 0, warnings: 0
        Elaborating the design hierarchy (primary partition):
        ...
        Writing primary snapshot: worklib.top:v
%
# Compile incremental.v and create incremental snapshot.
% xrun incremental.v -primname top

file: incremental.v
        module worklib.incremental:v
                errors: 0, warnings: 0
        ...
        Elaborating the incremental partition:
        ...
  assign status=primary.status;
                             |
xmelab: *N,HREFAC (./incremental.v,4|29): Hierarchical reference to primary partition requires permissions.
    The referenced object needs permission to be read or waited on.
    ...
xrun: *E,ELBERR: Error during elaboration (status 2), exiting.

These special permissions can be specified:

  • In a hierarchical reference permission file (href file). An href file is specified using the -href option when you create a primary snapshot.
    See The Hierarchical Reference Permission (href) File for details.
  • Directly as an argument to the -href option.
    See Specifying Hierarchical Reference Permissions with the -href Option.

The hierarchical reference permissions discussed in this chapter are different from the read, write, and connectivity access given to simulation objects by the -access  or -afile  options. The access permissions specified with the -access option or in an access file are for external references from outside the engine, while hierarchical permissions are for cross-snapshot references within the model. In some cases, a hierarchical reference may require setting external access permissions on its target object. These permissions can also be set in an href file. See Specifying External Access Permissions.
