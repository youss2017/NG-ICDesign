Specifying Hierarchical Reference Permissions with the -href Option

Hierarchical reference permissions can be specified directly as an argument to the -href option. The syntax is as follows:

-href "@absolute_path[...][<KEY>] permission"

The permission specification following the @ character is in the same format as a line in a hierarchical reference file.

White space is allowed before … and before <KEY>.

Examples:

# Full permission to objects in dut
-href "@dut *"
# Full permission to signals and ports in VHDL top
-href "@: ... *"
# Full permission to objects in dut.r1
-href "@dut.r1 *"
# All relevant permission to reg type objects in dut.r1
-href "@dut.r1<REG> *"
# All relevant permission to reg type objects in dut.r1 and below
-href "@dut.r1 ...<REG> *"
# Read and Write permission to reg type objects in dut.r1 and below
-href "@dut.r1 ...<REG> RW"
# Full permission to top.a till depth 4
-href "@dut.r1 ...4 *

You can use multiple -href options. For example:

% xmelab -mkprimsnap -href "@top ...<WIRE> *" -href "@top.a.b<REG> R" ....
