Supported SystemVerilog/UVM-SystemVerilog Ports

The following SystemVerilog/UVM-SystemVerilog ports are supported.

  • SystemVerilog Interface Ports
  • SystemVerilog Variable Ports

SystemVerilog Interface Ports

At a primary-instantiates-incremental boundary, the boundary port list can include SystemVerilog interface ports and modports. The interface is instanced in the primary partition and passed through the port to the incremental partition, either directly or via a modport.

At primary-instantiates-primary boundaries, and at incremental-instantiates-primary boundaries, interface ports are also supported, but with the following limitations:

  • Parameterized interface port with type parameters
    • Test case:

      //msie.sv
      interface intf #(type T = int)(input a);
      T intf_int;
      endinterface

      module prim(intf ifc);
      endmodule

    • Command line:

      xrun msie.sv -primtop prim

    • Error message:

      xmelab: *E,DYNIFP (./msie.sv,1|22): Interface port at the top of primary partition has unsupported parameter data type.

  • Parameterized interface with parameter data types of array of strings and union
    • Test case:

      //msie.sv
      interface intf #(string sa[2]={"M","S"})();
      endinterface

      module prim(intf ifc);
      endmodule

      module incr;
      intf #({"I", "E"}) intf_inst ();
        prim prim(.ifc(intf_inst));
      initial $display(prim.ifc.sa[1]);
      endmodule

    • Command line:

      xrun msie.sv -primtop prim -top incr -clean

    • Error Message:

      interface intf #(string sa[2]={"M","S"})();
               |
      xmelab: *E,DYNIFP (./msie.sv,1|25): Interface port at the top of primary partition has unsupported parameter data type.

  • Primary interfaces at the ports of a primary partition
    • Test case:

      // msie.sv
      interface intf();
      endinterface

      module prim(intf ifc);
      endmodule

      module incr;
      intf intf_inst ();
        prim prim(.ifc(intf_inst));
      endmodule

    • Command line:

      xrun msie.sv -primtop prim -top incr -clean -primtop intf

    • Error Message:

      prim prim(.ifc(intf_inst));
               |
      xmelab: *E,DYNIPN (./msie.sv,9|10): Interface port at the top of primary partition is itself a primary

  • Interface port having ports of interface type itself
    • Test case:

      //msie.sv
      interface intf_l(input clk);
      endinterface

      interface intf_u(intf_l intf_l_inst_1);
      endinterface

      module prim(intf_u ifc);
      endmodule

      module incr;
      logic clk;
      intf_l intf_l_inst_2(clk);
      intf_u intf_inst(.intf_l_inst_1(intf_l_inst_2));
        prim prim(.ifc(intf_inst));
      endmodule

    • Command line:

      xrun msie.sv -primtop prim -top incr -clean

    • Error Message:

      interface intf_u(intf_l intf_l_inst_1);
               |
      xmelab: *E,DYNIIP (./msie.sv,4|36): Interface port at the top of primary partition itself has interface port type.
      xrun: *E,ELBERR: Error during elaboration (status 1), exiting.

  • Interface port at the partition boundary which has instance hierarchy underneath
    • Test case:

      //msie.sv
      interface intf_l();
      logic clk;
      endinterface

      interface intf_u();
      intf_l intf_l_inst();
      endinterface

      module prim(intf_u ifc);
      endmodule
      module incr;
      intf_u intf_inst ();
      prim prim(.ifc(intf_inst));
      endmodule

    • Command line:

      xrun msie.sv -primtop prim -top incr -clean -primtop intf

    • Error Message:

      intf_l intf_l_inst();
      |
      xmelab: *E,DYNIHN (./msie.sv,6|17): Interface port at the top of primary partition has instance hierarchy underneath.

To implement interface ports at the top of a primary partition, a dummy instance of the interface is generated into an extern file (see Resolve Hierarchical References that Originate from a Primary). This dummy instance is associated with the port when the primary snapshot is elaborated. During final elaboration, the objects in the dummy instance are "mirrored" with the objects in the actual instance that is passed to the primary port from the parent partition.

SystemVerilog Variable Ports

Input variable ports are supported for SystemVerilog, but there are limitations. These are:

  • Input variable ports across mixed language boundary are not supported.
  • Input variable ports where primary instantiates incremental or another primary are not supported (see Input Port connectivity limitations for Primary- Instantiates- (Incremental or Primary)).

Inout variable ports have limitations. For inout ports, a DSSUFL error is generated (see Inout Port connectivity limitations for Primary- Instantiates- (Incremental or Primary)).

Connecting a variable (reg) to an inout or output port at the boundary is not allowed. The elaborator generates a PCIONC error (for inout) and a DSSVAR error (for output) (see Output Port connectivity limitations for Primary- Instantiates- (Incremental or Primary)).

Unsupported SystemVerilog Port Types

The following SystemVerilog port types are not supported at the partition boundary:

  • Associative Array
  • Type parameter
  • String/Class/Union/Unpacked Struct
  • Real
  • Event
  • Chandle
  • Virtual Interface
  • Dynamic/Unpacked Array
  • Queue
  • Mailbox
  • Semaphore

The elaborator generates an INCUSC error if unsupported port connection types are used.
