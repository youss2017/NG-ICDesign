Using Cold Run Flow

Complete the following steps to use the cold run flow:

  1. Use the -msieunlock cw_intf_optm option to build the primaries along with the existing href files. This optimizes all the interfaces during elaboration.

  2. Use the -msieunlock cw_intf_optm option to build the incremental. A sanity check would run an incremental build to test whether the optimized interfaces hold true. If any virtual interface declaration is found corresponding to the optimized interface, the optimization does not hold true, and pessimism has to be applied to that interface.
    The sanity checker would generate an autohref file with specific V entries, denoting the presence of a virtual interface declaration for the mentioned interface specialization. Affected primaries with such interfaces have to be built again with generated autohref files along with original href files if any.

    The extra 'V' entry in the autohref file denotes an interface specialization with a virtual interface declaration in the design. This information applies pessimism on an interface specialization during primary or incremental build in the warm run.
