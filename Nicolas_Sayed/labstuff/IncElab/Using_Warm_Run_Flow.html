Using Warm Run Flow

Complete the following steps to use the warm run flow:

  1. Build the primaries with the -msieunlock cw_intf_optm option, the autohref file (generated in the cold run) and, any existing href file. All the interfaces except the entries in the autohref file would be optimized during elaboration.
  2. Build the incremental with the -msieunlock cw_intf_optm option.

    1. The incremental build would run a sanity checker, ensuring all the interface instances with a virtual interface (VIFC) declaration have been applied pessimism. When sanity passes, the incremental build finishes successfully.

    2. In case the sanity checker detects that some of the interface instances are not applied pessimism and have compatible vifc declarations (may have been added recently), it would generate autohref with 'V' entry again and prompt you to re-elaborate the affected primaries, hence converting the current warm run into a cold run phase.

    3. If the sanity checker detects that some of the interface instances are applied pessimism and have no compatible vifc declarations (may have been deleted recently), it would warn the user having primaries with non-optimized interfaces unnecessarily and advise to re-elaborate those primaries to get those interfaces optimized.
