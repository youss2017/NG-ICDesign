Verilog and SystemVerilog Datatypes Supported for Hierarchical References

Hierarchical references are supported for Verilog and SystemVerilog. The following Verilog and SystemVerilog data types are supported for hierarchical references:

  • Verilog 4-state, 2-state, real, event, and vector data types and unpacked arrays of these data types.
  • SystemVerilog bit and shortreal data types and unpacked arrays of these data types.
  • Verilog and SystemVerilog tasks and functions with scalar or vector arguments of type register or type string.
    • Arrays of data types are not supported as arguments to task and function.

    • Arguments of a target task or function cannot have variables or identifiers as their default arguments. Only constants are supported as the default arguments for the target task or function.

      Although hierarchical references to tasks/functions with string type arguments are supported, hierarchical references to the string type, in general, are not supported.

  • Parameters as targets of a hierarchical reference.
    • Parameters can be read, but not written to. (This means that a defparam cannot set the value of a target parameter in another partition.)
    • References to parameters of complex SystemVerilog data types such as packed structs, unpacked structs, or unpacked arrays are not supported.

All other data types are not supported.

For unsupported scenarios, the elaborator generates a CUHPNM error.

*E,CUHPNM: Illegal pathname element <InstanceName> - unable to resolve external name. [File:<file name>, Line:<line>].
